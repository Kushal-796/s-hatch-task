## 🎯 How It Works

### **1. User Journey**

1. **Landing Page** → Beautiful hero with typewriter animation showcasing platform benefits
2. **Registration/Login** → Firebase Auth with Google OAuth or email/password
3. **Dashboard Access** → Protected route with AI chat interface
4. **Feature Usage** → Plan-based access with real-time usage tracking
5. **Upgrade Flow** → Seamless Razorpay payment integration for plan upgrades

### **2. Technical Architecture**

- **Frontend**: React 19 with Vite for fast development and building
- **Styling**: Tailwind CSS with custom dark theme and animations
- **Backend**: Firebase (Authentication + Firestore database)
- **Payments**: Razorpay integration for Indian market
- **State Management**: React Context API for auth and plan management
- **Routing**: React Router v6 with protected routes
- **Animations**: Framer Motion for smooth interactions

### **3. Business Logic**

- Users start with Free plan (3 uses)
- AI assistant usage increments usage counter
- Visual usage meters show remaining quota
- Upgrade prompts appear when limits reached
- Payment processing updates user plan instantly
- Real-time plan enforcement across the app

## ⚙️ Quick Setup Guide

### **Prerequisites**

- Node.js 18+
- npm/yarn package manager
- Firebase project (free tier sufficient)
- Razorpay account (for payment integration)

### **1. Installation**

```bash
git clone https://github.com/Kushal-796/s-hatch-task.git
cd s-hatch-task
npm install
```

### **2. Environment Configuration**

Create `.env.local` in the project root:

```env
# Firebase Configuration (Replace with your actual values)
VITE_FIREBASE_API_KEY="your_firebase_api_key"
VITE_FIREBASE_AUTH_DOMAIN="your_project_id.firebaseapp.com"
VITE_FIREBASE_PROJECT_ID="your_project_id"
VITE_FIREBASE_STORAGE_BUCKET="your_project_id.appspot.com"
VITE_FIREBASE_MESSAGING_SENDER_ID="your_sender_id"
VITE_FIREBASE_APP_ID="your_app_id"

# Razorpay Configuration (Test Mode)
VITE_RAZORPAY_KEY_ID="rzp_test_your_razorpay_key_id"
```

### **3. Firebase Setup**

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com)
2. Enable Authentication → Google + Email/Password providers
3. Create Firestore database with test mode rules
4. Copy configuration values to `.env.local`

### **4. Razorpay Setup**

1. Sign up at [razorpay.com](https://razorpay.com)
2. Get test API keys from dashboard
3. Add test key to `.env.local`
4. Configure webhook URLs for production

### **5. Start Development**

```bash
npm run dev
# Opens at http://localhost:5174
```

```
└── README.md               # This file
```

### **Key Architecture Decisions**

#### **🔧 State Management Strategy**

- **AuthContext**: Handles Firebase auth, user data, and session management
- **PlanContext**: Manages subscription plans, usage tracking, and payments
- **React Router**: Client-side routing with authentication guards
- **Context Providers**: Wrap entire app for global state access

#### **🎨 Styling Architecture**

- **Tailwind CSS**: Utility-first with custom theme configuration
- **Custom CSS**: Glass morphism effects, animations, and gradients
- **Component-scoped**: Styles defined per component for maintainability
- **Responsive Design**: Mobile-first breakpoints throughout

#### **🔐 Security Implementation**

- **Protected Routes**: Authentication required for dashboard access
- **Firebase Security Rules**: Server-side data access control
- **Environment Variables**: Sensitive keys stored securely
- **Input Validation**: Form validation and sanitization

#### **💳 Payment Integration**

- **Razorpay SDK**: Dynamic script loading for optimal performance
- **Plan Definitions**: Centralized pricing configuration in razorpay.js
- **Usage Tracking**: Real-time Firestore updates for usage limits
- **Error Handling**: Comprehensive payment failure management
